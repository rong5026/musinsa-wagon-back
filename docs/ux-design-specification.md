# UX Design Specification - musinsa-wagon-nextjs-react-ts-v2

**Author:** Hong-yeonghwan
**Date:** 2025-12-06

---

## Executive Summary

### Project Vision

**Musinsa Wagon**은 20~30대 패션 소비자들이 "가짜 할인"의 배신감에서 벗어나 **"효율적이고 똑똑한 소비"의 만족감**을 경험하게 하는 가격 투명성 플랫폼입니다.

**핵심 감정 전환:**

- Before: 명절 세일의 배신감, "또 속았구나"
- After: 데이터 기반 확신, "이제 진짜 할인을 알아"

**사용 맥락:**

- 명절 전 가격 비교 피크 타임
- 출퇴근 시간 쇼핑몰 탐색 중
- 마음에 드는 상품 발견 → 즉시 가격 검증 → 구매 판단

**차별점:**

- 단순 가격 추적이 아닌 "**지금 사도 되는지**" 판단 지원
- 명절 속임수 탐지로 **배신감 → 안도감** 전환
- 무신사/에이블리/지그재그 통합 추적으로 **시간 절약**

### Target Users

**Primary: 지현 (28세, IT 직장인)**

- 출퇴근 지하철에서 패션 앱 탐색
- "똑똑한 소비"에 관심, 절약 인증을 친구들과 공유
- 경험: 명절 세일에 속아본 경험 → 의구심 생김
- 니즈: "이 가격이 진짜 할인인지 확인하고 싶어"
- 행동: 무신사 좋아요 + 경험에 의존 (불확실)
- 기대: 데이터로 증명된 "지금 사세요" vs "기다리세요"

**Secondary: 민수 (29세, 개발자)**

- 기술 스택에 관심 + 실용성 추구
- GitHub에서 프로젝트 발견 → 직접 사용 → 피드백 제공
- 니즈: "실제로 쓸 만한 도구면서 코드도 배울 수 있는 것"
- 행동: 얼리어답터, 커뮤니티 공유, 기여 의지
- 기대: 깔끔한 UI/UX + 기술적 완성도

**공통 특성:**

- 20~30대, 모바일 중심 사용
- 푸시 알림에 익숙함
- SNS/커뮤니티 활동 활발 (바이럴 가능성)
- "속지 않는 소비자"로서의 자부심 추구

### Key Design Challenges

**1. 마찰 최소화: URL 복붙의 번거로움**

- 문제: 쇼핑몰 → Musinsa Wagon → URL 복사 → 붙여넣기 과정이 번거로움
- MVP 접근: 명확한 안내 + 빠른 피드백 (등록 즉시 결과 표시)
- Post-MVP: 브라우저 확장 프로그램으로 원클릭 등록

**2. 즉시성: 구매 결정 순간의 빠른 판단**

- 사용자는 "지금 당장" 사야 할지 알고 싶어함
- 복잡한 그래프보다 **명확한 라벨** (🟢 역대 최저가 / 🔴 비싼 편)
- 3초 이내에 판단 가능한 시각적 명확성 필요

**3. 신뢰 구축: "이 데이터 믿어도 돼?"**

- 사용자는 처음엔 의심할 수 있음
- 투명한 가격 히스토리 그래프로 신뢰 구축
- 명절 속임수 탐지 알림으로 "이 서비스가 나를 보호해준다" 경험 제공

**4. 모바일 최적화**

- 출퇴근 시간 사용 = 한 손 조작, 작은 화면
- 복잡한 인터랙션보다 간단명료한 정보 전달
- 빠른 로딩 + 오프라인 캐싱 고려

### Design Opportunities

**1. 감정 중심 UX: 배신감 → 안도감 전환**

- 가짜 할인 경고 알림 = 사용자를 "보호"하는 경험
- "속지 않았어"라는 안도감 + "똑똑한 소비자"라는 자부심
- 이 감정 전환이 바이럴의 핵심 (친구에게 자랑하고 싶은 경험)

**2. 즉시 판단 가능한 시각 언어**

- 색상 라벨 (🟢🟡🔴) + 명확한 카피
- "지금 사세요" vs "조금 기다리세요" 같은 액션 가이드
- 그래프는 보조 수단, 라벨이 주 판단 기준

**3. 알림의 가치 극대화**

- "역대 최저가 갱신!" = 즉시 구매 동기 부여
- "가짜 할인 의심" = 충동 구매 방지
- 알림 하나하나가 사용자에게 **금전적 가치** 제공

**4. 바이럴 친화적 UX**

- 절약 금액 표시 (한 달에 10만 원 절약!)
- SNS 공유 가능한 스크린샷 (가짜 할인 탐지 증거)
- GitHub 오픈소스 접근성 (개발자 커뮤니티 확산)

## Core User Experience

### Defining Experience

**핵심 사용자 루프: "찜 → 추적 → 알림 → 구매 판단"**

Musinsa Wagon의 핵심 경험은 **수동적 가격 비교에서 능동적 가격 추적**으로의 전환입니다. 사용자는 단순히 "지금 얼마인가?"를 확인하는 것이 아니라, "언제 사야 가장 좋은가?"를 데이터 기반으로 판단합니다.

**가장 빈번한 사용자 행동:**

1. **찜 목록 확인** - 출퇴근 시간에 추적 중인 상품들의 가격 변동 스캔
2. **알림 확인** - "역대 최저가" 또는 "가짜 할인 의심" 알림 수신 시 즉시 반응
3. **새 상품 등록** - 쇼핑몰에서 마음에 드는 상품 발견 시 URL 복사 → 붙여넣기

**제품의 가치를 정의하는 핵심 상호작용:**

- **가격 평가 라벨 확인** (🟢 역대 최저가 / 🟡 평균 가격 / 🔴 비싼 편)
- **가격 히스토리 그래프 탐색** - 명절 가격 조작 패턴 시각화
- **알림 기반 즉시 행동** - "지금 사야 해" 순간의 마찰 제거

### Platform Strategy

**Primary Platform: Mobile Web (PWA)**

출퇴근 시간 지하철에서의 사용을 최우선으로 고려한 모바일 웹 우선 전략:

**플랫폼 결정 근거:**

- ✅ **설치 장벽 제거** - 앱 스토어 거치지 않고 즉시 사용 가능
- ✅ **크로스 플랫폼** - iOS/Android 동시 지원, 개발 리소스 효율화
- ✅ **PWA 장점 활용** - 홈 화면 추가, 푸시 알림, 오프라인 캐싱
- ✅ **빠른 업데이트** - 앱 스토어 승인 없이 즉시 배포

**인터랙션 패턴:**

- **터치 기반 설계** - 한 손 조작 최적화 (엄지손가락 도달 범위 고려)
- **스와이프 제스처** - 찜 목록에서 좌우 스와이프로 상품 관리
- **Pull-to-refresh** - 실시간 가격 업데이트 확인

**오프라인 기능:**

- 캐시된 찜 목록 및 가격 히스토리 조회
- 네트워크 복구 시 자동 동기화
- "오프라인 상태" 명확한 시각적 피드백

### Effortless Interactions

**"3초 판단 원칙" - 생각 없이 즉시 이해 가능**

**1. 찜 목록 스캔 = 색상 라벨만 보기**

- 🟢 **초록색 뱃지** = "지금 사세요" (즉시 행동 유도)
- 🔴 **빨간색 뱃지** = "기다리세요" (충동 구매 방지)
- 🟡 **노란색 뱃지** = "평범해요" (중립적 정보)

라벨만 스캔하면 10개 상품의 구매 타이밍을 **3초 이내에 판단** 가능.

**2. 알림에서 행동까지 원클릭**

- 알림 탭 → 즉시 해당 상품 상세 페이지로 이동
- "지금 구매하기" 버튼 → 원본 쇼핑몰로 자동 이동 (URL 복사 불필요)
- 알림 자체가 이미 **행동 가능한 정보** (예: "역대 최저가 -25% 갱신!")

**3. 자동 감지 = 사용자 개입 불필요**

- **가짜 할인 자동 탐지** - 사용자가 패턴 분석할 필요 없음
- **가격 변동 자동 추적** - 매일 수동 확인 불필요
- **최적 구매 타이밍 알림** - 사용자가 판단하지 않아도 앱이 알려줌

**4. URL 등록의 단순화 (MVP 한계 인정)**

- 현실: URL 복사-붙여넣기는 번거로움
- 완화책: 등록 즉시 가격 분석 결과 표시 (보상으로 마찰 상쇄)
- 미래: 브라우저 확장 프로그램으로 원클릭 등록 (Post-MVP)

### Critical Success Moments

**"Wow Moment" = 역대 최저가 알림 수신**

**첫 번째 성공 순간:**
사용자가 찜해둔 상품이 처음으로 **역대 최저가를 갱신**하고, 푸시 알림을 받는 순간:

> "🎉 [상품명] 역대 최저가 갱신! -25% → -35%
> 지금 바로 확인하세요!"

**이 순간이 중요한 이유:**

1. **금전적 가치 증명** - "이 앱이 내 돈을 아껴준다"는 직접적 경험
2. **신뢰 구축** - 데이터 기반 추천의 정확성 체감
3. **바이럴 트리거** - 친구들에게 자랑하고 싶은 경험 ("나 이 앱으로 5만 원 아꼈어!")

**두 번째 성공 순간: 가짜 할인 탐지**

명절 전 쇼핑몰에서 "50% 할인!"을 보고 등록했는데, 앱이 경고 알림:

> "⚠️ 가짜 할인 의심
> 평소 가격: 89,000원 → 할인가: 89,000원
> 실제 할인율: 0%"

**감정 전환:**

- Before: "또 속을 뻔했네..." (배신감)
- After: "이 앱이 날 지켜줬어!" (안도감 + 신뢰)

**세 번째 성공 순간: 첫 상품 등록 완료**

새 사용자가 처음 URL을 붙여넣고 **3초 이내에 가격 분석 결과**를 보는 순간:

- 즉시 가격 히스토리 그래프 표시
- 현재 가격 평가 라벨 (🟢🟡🔴)
- "지금 사세요" 또는 "조금 기다리세요" 명확한 가이드

**Make-or-Break 플로우:**

1. **온보딩 → 첫 상품 등록 → 즉시 가치 경험** (5분 이내)
2. **찜 목록 확인 → 색상 라벨 스캔 → 3초 판단** (매일 반복)
3. **알림 수신 → 원클릭 행동** (구매 전환의 핵심)

### Experience Principles

**UX 설계의 4가지 핵심 원칙:**

**1. "3초 판단" 원칙**

- 모든 정보는 3초 이내에 이해 가능해야 함
- 색상, 라벨, 명확한 카피로 즉시 판단 지원
- 그래프와 상세 정보는 **보조 수단**, 라벨이 **주 판단 기준**

적용 예시:

- ✅ 찜 목록: 색상 라벨만 스캔 → 10개 상품 3초 판단
- ❌ 복잡한 통계 표 → 사용자 분석 부담

**2. "보호받는 느낌" 원칙**

- 사용자를 "속임수"로부터 보호하는 경험 설계
- 가짜 할인 탐지 = 앱이 사용자 편에 서 있다는 증명
- 알림은 "판매 촉진"이 아닌 **"사용자 이익 보호"** 목적

적용 예시:

- ✅ "가짜 할인 의심" 경고 알림 (충동 구매 방지)
- ❌ "지금 사세요!" 무분별한 구매 압박

**3. "노력 없는 추적" 원칙**

- 자동화로 사용자 개입 최소화
- 한 번 등록하면 앱이 알아서 추적 → 알림
- 사용자는 판단만 하면 됨 (분석은 앱이 담당)

적용 예시:

- ✅ 자동 가격 추적 + 최저가 알림
- ❌ 매일 수동으로 가격 확인 필요

**4. "가치 전달 우선" 원칙**

- 첫 5분 안에 사용자가 "이 앱 쓸 만한데?"를 느껴야 함
- 온보딩보다 **즉시 가치 경험** 우선
- 알림 하나하나가 금전적 가치 제공

적용 예시:

- ✅ 첫 상품 등록 즉시 가격 분석 결과 표시
- ❌ 긴 튜토리얼 후 실제 기능 사용
